Image to 3D Gaussian Splatting

i23DGS 商品图转高斯展示平台

i23DGS 是一个围绕商品图片、主体抠图、3DGS 模型生成桥接和 Web 端交互展示的实验工作台。 它把 AI 图像处理、后端流程编排和 Three.js 高斯渲染串成一条可运行的工程链路。

打开工作台 查看 README
3DGS 不是传统三角面模型，而是一组带位置、尺度、旋转、颜色和透明度的 3D 高斯点。

当前 Demo 做了什么

图像预处理

上传商品图片后，Node.js API 调用 Python rembg 输出透明 PNG，作为后续模型生成输入。

模型生成桥接

本地 baseline 会从透明 PNG 构建 2.5D .splat；真实工业链路可切换为远端 AI Provider。

Web 端渲染

前端使用 React、Three.js 和 GaussianSplats3D 加载 .splat/.ksplat/.ply，支持旋转、缩放、播放和录制。

流水线

1. Upload 用户上传带背景的商品图，例如耳机、奶茶、鞋、包或小家电。
2. Matting Python rembg 提取主体，输出带透明通道的 PNG。
3. Generate 本地脚本生成 2.5D splat baseline；工业模式下把透明 PNG 发给 LGM、TripoSR、多视图扩散或商业 3D API。
4. Render 浏览器加载高斯模型，围绕模型中心播放一圈，必要时通过 MediaRecorder 导出 WebM。

本地 baseline 与真实 3DGS 的区别

类型 输入 原理 适合用途
本地图像高斯 baseline 1 张图片 把透明 PNG 的像素采样成彩色 Gaussian，并给一个浅层 2.5D 深度。 验证工程链路、快速预览、前端加载测试。
真实 3D Gaussian Splatting 几十到几百张图片或视频抽帧 通过相机位姿估计、稀疏点云初始化和高斯优化训练出完整 3D 场景。 高质量自由视角展示、空间重建、沉浸式浏览。
单图转 3DGS AI 1 张图片 大模型预测不可见角度和几何结构，再输出 mesh 或 Gaussian 表示。 商品快速建模、低成本资产草稿，但质量依赖模型能力。

需要多少图片

如果目标是工业级商品重建，建议围绕物体拍摄上、中、下三层角度，每层 24-48 张， 总量大约 72-144 张。场景级重建通常需要 150-500 张以上。 单图方案可以跑通，但背面、侧面和遮挡区域主要依赖 AI 推断。
