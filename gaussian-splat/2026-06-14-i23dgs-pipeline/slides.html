i23DGS / image to 3D gaussian splatting

把一张商品图接进可演示的 3DGS 工程流水线

这不是“论文训练器”，而是一个前端工程师能完整跑通的实验工作台：上传图片、自动抠图、生成或接入高斯模型，再在浏览器里查看、运镜和录制。

React + Three.js Node TypeScript API Python rembg .splat / .ksplat / .ply

主样例选用黑色头戴耳机：主体清晰、材质高级、结构有弧线和厚度，适合展示从商品图到高斯资产的完整链路。

Thesis

项目价值在于打通链路，而不是假装本地脚本已经等于工业级 3D 重建。

给前端的价值

从图片输入到 WebGL/Three.js 展示全链路可控，便于验证交互、加载、录制和资产管理。

给 AI 流水线的价值

Provider 接口已经抽象出来，本地 baseline 与远端工业模型可以复用同一套前后端状态机。

给产品演示的价值

可上传商品图，可加载真实样例模型，也可把相机动画录成 WebM，用于快速说明效果边界。

一句话：i23DGS 是“商品 3D 资产生成”的工程骨架，先把每个接口、状态和前端体验跑顺，再替换更强的 3DGS Provider。

定位：工程桥，而不是最终模型能力声明02

Pipeline

主流程是五段：上传、抠图、生成、加载、录制。

01 Upload 用户选择商品图，API 保存到 data/uploads 并创建 job。
02 Matting Node 调 Python rembg，输出透明主体 PNG。
03 Provider 本地脚本生成 2.5D .splat，或切到远端 AI 服务。
04 Viewer Three.js + GaussianSplats3D 加载模型并重置相机。
05 Record 相机绕物体中心运动，Canvas MediaRecorder 导出 WebM。
运行：npm install npm run mock:splat npm run dev
从“文件上传”到“可分享视频”的最短闭环03

Provider Boundary

本地 baseline 与远端 Provider 共用同一个接口。

mock-local

调用 scripts/image_to_splat.py，从透明 PNG 的像素采样出一份真实可加载的 .splat 文件。

适合验证工程链路、前端加载、动画导出和 UI 反馈。

remote-api

把透明 PNG 作为 image/png 请求体发给外部模型服务，返回 .splat 或 .ply。

适合接商业 3D API、LGM/TripoSR 类模型或自建多视图生成服务。

远端配置：$env:I23DGS_AI_ENDPOINT="https://your-provider.example.com/i23dgs" $env:I23DGS_AI_KEY="..."
抽象边界：generate(inputPngPath, jobId) => outputPath + format06

Quality Truth

要讲清楚三种“看起来像 3D”的东西，不然很容易误会。

类型 输入 真实能力 在项目中的角色
本地图像高斯 baseline 单张透明 PNG 把图像像素变成带浅层深度的 splats，能旋转但背面信息不是真重建。 跑通文件、状态、Viewer、录制和 Provider 接口。
真实 3DGS 场景 多图或视频抽帧 通过相机位姿、点云初始化和高斯优化重建空间。 通过官方样例 .ksplat 验证加载与交互质量。
单图转 3D AI 一张商品图 模型推断不可见面，再输出 mesh、Gaussian 或其他 3D 表示。 未来替换 remote Provider 的主要方向。
边界说清楚，项目可信度会上升07

Viewer Experience

前端不是只“显示模型”，而是在构建可讲故事的商品展示台。

3 支持 .splat、.ply、.ksplat 三类模型输入。
7.2s 默认相机轨道动画时长，围绕当前模型中心播放一圈。
WebM 直接录制 Canvas 输出，方便把交互结果变成可分享视频。

体验闭环

加载默认耳机、上传商品图、选择真实场景、拖入外部模型、播放运镜、录制结果，这些动作覆盖了商品 3D 展示最常见的演示路径。

演示重点：从模型能力，转成“前端可消费的内容资产”08

Demo Talk Track

现场讲解可以按“先可信、再惊喜、最后扩展”的顺序走。

建议演示顺序

  1. 打开默认耳机，说明 Viewer 和相机控制。
  2. 上传一张商品图，展示 job 进入抠图。
  3. 查看透明 PNG，说明主体提取。
  4. 点“本地图像高斯”，加载 baseline .splat。
  5. 播放运镜并录制 WebM。
  6. 切换真实 3DGS 样例，强调未来 Provider 接入空间。

推荐话术

“本地版本解决的是工程闭环，不负责承诺工业建模质量。真正上线时，把 Provider 换成多图/单图 3D 服务，前端、API、状态机和录制工作流都可以保留。”

这句话能同时保护项目边界，也能让听众看到它的可扩展价值。
