minimal-viewer
怎么搭 & 3DGS 怎么画

单文件 WebGL2 Demo：从 32 字节点云，到屏幕上三团柔光。

按 → 翻页 · Live Demo：index.html

这个 Demo 是什么？

  • 不是 游戏模型（没有三角面、贴图）
  • 不是 视频或全景图
  • 是 3D Gaussian Splatting 的最小浏览器 Viewer

用 3500 颗半透明模糊椭球叠出画面。Demo 里是程序生成的红/绿/蓝三团，也可拖入真实 .splat 文件。

和传统 3D 有何不同？

  • 基本单元：三角形+纹理 vs 3D 高斯椭球+颜色
  • 场景来源：人工建模 vs 多张照片 AI 重建
  • 深度：Z-buffer vs 按深度排序 + Alpha 混合
  • 观感：硬边清晰 vs 软边柔光、会渗色

优点：拍就能重建、转视角自然。缺点：文件大、难编辑，更像「冻结的光场」。

一颗高斯存什么？32 字节

偏移 0–11   float32 × 3   位置 XYZ
偏移 12–23  float32 × 3   缩放 scale（椭球多大）
偏移 24–27  uint8 × 4     颜色 RGBA
偏移 28–31  uint8 × 4     旋转四元数（椭球朝向）

3500 点 × 32B ≈ 109 KB。createMockSplat() 在内存里直接生成，不必下载大文件。

Demo 四层结构

  • UI — HUD · 8 段导览
  • 数据 — Mock / 拖入 .splat
  • Worker — 纹理打包 · 排序
  • WebGL2 — Shader 绘制 · 混合

全部在一个 index.html，无 npm。渲染核心 fork 自 antimatter15/splat（MIT）。

主线程 vs Worker

主线程

  • WebGL2 画屏
  • 相机 / 导览 / 输入
  • 每帧 drawArraysInstanced

Web Worker

  • generateTexture() — .splat → GPU 纹理
  • runSort() — 按深度排序
  • postMessage 回传 texdata + depthIndex

半透明不能靠 Z-buffer，必须从远到近画，排序在 CPU 做。

每帧发生了什么？

  1. 更新相机 viewMatrix
  2. 把 view 发给 Worker → 重新排序
  3. Worker 回传排序后的 index 缓冲
  4. clearColor(0,0,0,0) — alpha 必须是 0
  5. 实例化绘制 3500 个 quad（每颗高斯一个）

左上角 fps 旁的 splats 数 = 当前参与绘制的点数

3DGS 渲染管线（原理）

3D 椭球 → 投影到屏幕 → 3D 协方差变 2D 协方差 → 画 2D 椭圆 quad → exp 衰减 + Alpha 叠加

每颗高斯在屏幕上不是画一个点，而是画一张随视角变形的 2D 模糊椭圆。

顶点 Shader 做什么？

  • 用 index 从整数纹理 texelFetch 读位置 & 协方差
  • 3D 协方差 → 投影 → 2D 协方差 cov2d
  • 算椭圆长轴 majorAxis、短轴 minorAxis
  • 同一个 quad 模板实例化 3500 次
gl_Position = vCenter + position.x * majorAxis/viewport
                        + position.y * minorAxis/viewport

片元 Shader + 混合

float A = -dot(vPosition, vPosition);
float B = exp(A) * vColor.a;
fragColor = vec4(B * vColor.rgb, B);
  • 中心亮、边缘虚 → 柔光
  • DEPTH_TEST 关闭，全靠排序
  • 混合：ONE_MINUS_DST_ALPHA, ONE

踩坑：若 clearColor 的 alpha=1，半透明叠不上去 → 全黑。

Worker：点云 → GPU 纹理

原始 .splat GPU 不能直接读，Worker 打包成 RGBA32UI 纹理：

  • Texel A — 位置 XYZ（float bits）
  • Texel B — 协方差（half-float）+ RGBA

每颗高斯占 2 个 texel。3500 点 → 约 2048×4 纹理。

为什么这样构建 Demo？

  • 单文件 — 一眼看完管线
  • Mock 三球 — 不依赖大场景文件
  • Worker 排序 — 和真 Viewer 一致
  • 8 段导览 — 相机对准红/绿/蓝
  • 可拖 .splat — 接真实数据

不做 React / Three.js，避免藏掉 Shader 和排序细节。

从拍照到屏幕（完整故事）

多张照片 → 训练（Demo 未做）→ 导出 .splat → Viewer 加载 → 排序 + Shader → 三团柔光

训练学出每颗高斯的参数；Viewer 只负责读参数、排序、画 2D 高斯斑、混合。

对照：看画面时想什么

  • 左红右绿下蓝 → Mock 三个 cluster
  • 边缘发虚 → 片元 exp(-r²) 衰减
  • 旋转前后关系对 → Worker 每帧 runSort
  • 3500 splats → drawArraysInstanced 实例数

建议阅读代码顺序

  1. createMockSplat — 数据从哪来
  2. createWorker → generateTexture / runSort
  3. 顶点 Shader VS — 3D→2D 椭球
  4. 片元 Shader FS + blendFuncSeparate
  5. frame() — 每帧串联
