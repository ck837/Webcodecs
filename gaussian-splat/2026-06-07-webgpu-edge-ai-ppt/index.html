核心解构：Shader 与并行计算

GPU 是成千上万个核心同时开工的并行机器。 Shader 是直接运行在 GPU 上的像素 / 顶点计算程序。

CPU Serial Loop

显式遍历屏幕，每个像素排队计算。吞吐被单线程调度、缓存局部性与主线程负载持续压制。

for y in height: for x in width: uv = vec2(x, y) / resolution color = shade(uv, time) framebuffer[x, y] = color

GPU Fragment Shader

没有全局 for 循环。每个像素 / 顶点同时独立执行同一段数学逻辑。

// 每个 Fragment 并行执行 fn fragment(uv, t) -> color { field = noise(uv * 8.0 + t) return emit(field, uv) }

核心心智：输入 UV 坐标与时间 t，GPU 在一个帧周期内把数学场喷刷成屏幕。

核心解构：WebGPU / WGSL

WebGPU 是显式图形与计算 API，直接映射现代 GPU 硬件。 WGSL 是它的专用着色器语言。
  • 与 WebGL 的代差：解除 CPU 单线程状态机瓶颈，支持多线程并行提交命令。
  • 原生支持 Compute Shader：渲染管线之外，直接调度通用并行计算。
  • 端侧 AI 商业价值：不仅能画画，更能在端侧执行 LLM / VLM 矩阵乘法并行加速。
  • 算力成本转移：推理从云端 GPU 下沉到用户终端，企业侧获得接近 零算力成本 的交付模型。

核心解构：Buffer / Bind Group

Buffer 是显存中的连续内存块。 Bind Group 是 CPU 与 GPU 之间的显存指针和内存布局接口契约。

Memory Contract

  • 顶点、索引、Uniform、Storage 全部落到 Buffer。
  • Bind Group 描述资源槽位、访问权限、着色器可见性。
  • 核心卡点是 Memory Alignment，例如 16 字节对齐。

Performance Deadline

  • 流式音视频、动态点云、传感器输入，本质都是跨硬件数据搬运。
  • 精通 Buffer 才能做 Zero-Copy，直接送入 GPU。
  • 避免主线程阻塞，避免掉帧，避免交互链路被内存拷贝拖死。

降维交集：Three.js / WebXR 的真实关系

Three.js / WebXR 是应用层外壳：场景挂载、双眼对齐、相机控制。 Shader / WebGPU / Buffer 才是核心发动机。
  1. Three.js 的 ShaderMaterial 承载 AI 神经渲染，例如 3DGS 的像素输出。
  2. 新版 Three.js 正在全量重构 WebGPURenderer，它会成为未来 5 年的底层核心。
  3. WebXR 触碰 90Hz / 120Hz 眩晕死线，必须靠 Buffer 优化和显存控制确保不掉帧。
