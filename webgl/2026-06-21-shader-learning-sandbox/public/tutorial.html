1. 这 Demo 要干什么

这是一个最小但完整的自定义 Shader 学习环境：一个会转的球体，颜色由你自己写的 GLSL 决定，而不是 Three.js 内置的 MeshStandardMaterial。

你要建立的核心心智：

React 管「什么时候、传什么数」（Uniform、旋转）
GLSL 管「每个像素算什么颜色」（Fragment Shader）
两者通过 ShaderMaterial 绑在一起。

改 src/shaders/fragmentShader.ts 里的一行数学，保存后浏览器立刻变样 — 这是学 Shader 最快的方式。

2. GPU 渲染流水线（一图看懂）

每一帧，GPU 大致做这些事：

几何体顶点 (position, uv, normal) ← Attribute，来自 SphereGeometry ↓ Vertex Shader（顶点着色器） ← vertexShader.ts · 3D 位置 → 屏幕位置 gl_Position · 写出 vUv、vNormal ← Varying，传给下一 stage ↓ 光栅化（GPU 硬件） ← 把三角形拆成像素，Varying 插值 ↓ Fragment Shader（片元着色器） ← fragmentShader.ts ★ 你主要改这里 · 每个像素算一次 gl_FragColor · 读 uTime、uMousePosition ← Uniform，来自 React ↓ 屏幕上的彩色球体

Vertex 跑在顶点上（球体约几千个顶点）；Fragment 跑在像素上（可能上百万次）。视觉效果主要在 Fragment 里做。

3. React Three Fiber 扮演什么角色

层文件干什么
React UIApp.tsx布局、Canvas、鼠标坐标、OrbitControls
R3F 场景ShaderSphere.tsxmesh + shaderMaterial + useFrame
GLSLshaders/*.ts真正在 GPU 执行的字符串
Three.js（底层）编译 GLSL、上传 Uniform、发 draw call

没有 R3F 也能写 WebGL，但要手写 scene/camera/renderer 循环。R3F 让你用 JSX 声明 3D，用 useFrame 挂钩渲染循环 — 和写 React 组件一样自然。

5. 三类 GLSL 变量（必背）

类型谁写谁读本 Demo 例子
Attribute 几何体 Vertex Shader position uv normal（Three 自动注入）
Uniform CPU / React Vertex 或 Fragment uTime uMousePosition modelViewMatrix
Varying Vertex Shader Fragment Shader（插值后） vUv vNormal
记忆口诀：Attribute 每顶点不同；Uniform 整帧相同；Varying 顶点写出、像素读到的是插值结果。

6. Vertex Shader 逐行讲

文件：src/shaders/vertexShader.ts

// 声明 Varying：会传给 Fragment Shader
varying vec2 vUv;
varying vec3 vNormal;

void main() {
  // Attribute uv → Varying vUv（球面经纬度参数化到 0~1）
  vUv = uv;

  // Attribute normal，经 normalMatrix 变到视图空间
  vNormal = normalize(normalMatrix * normal);

  // Attribute position × MVP → 屏幕裁剪坐标（必写 gl_Position）
  gl_Position = projectionMatrix * modelViewMatrix * vec4(position, 1.0);
}

本 Demo 的 Vertex Shader故意保持最小：只做标准变换 + 传递 UV/法线。复杂变形（波浪、溶解）也可以写在这里，改 position 再赋给 gl_Position。

7. Fragment Shader 逐行讲

文件：src/shaders/fragmentShader.ts — 视觉效果的核心。

Uniform 声明

uniform float uTime;           // React 每帧写入：过了多少秒
uniform vec2 uMousePosition; // React 写入：鼠标 UV 0~1
varying vec2 vUv;            // 来自顶点，像素级插值
varying vec3 vNormal;

① 彩虹波纹（时间 + UV）

float wave = sin(vUv.x * 10.0 + uTime * 2.0) * 0.5 + 0.5;

vUv.x 是横向坐标，uTime 让条纹随时间移动。改 10.0 → 条纹密度；改 2.0 → 动画速度。

② 鼠标热点

float dist = length(vUv - uMousePosition);
float mouseGlow = 0.35 / (dist * 6.0 + 0.08);

当前像素 UV 与鼠标 UV 的距离越近，mouseGlow 越大 — 演示 Uniform 交互。改 6.0 控制热点范围。

③ 法线明暗（体积感）

float ndotl = dot(normalize(vNormal), vec3(0.0, 0.0, 1.0)) * 0.5 + 0.5;

用 Varying vNormal 做假光照：朝向相机的面更亮。没有这一步，球面会像扁平色块。

④ 输出

gl_FragColor = vec4(finalColor, 1.0);

每个 Fragment 必须写 gl_FragColor（WebGL1 写法；WebGL2 可用 out vec4 fragColor）。

8. React → GPU：useFrame 桥梁

文件：src/components/ShaderSphere.tsx

// 1. 创建 Uniform 初值（只建一次）
const uniforms = useMemo(() => ({
  uTime: { value: 0 },
  uMousePosition: { value: new THREE.Vector2(0.5, 0.5) },
}), []);

// 2. 每帧更新 — R3F 在 requestAnimationFrame 里调用
useFrame((state) => {
  mat.uniforms.uTime.value = state.clock.elapsedTime;
  mat.uniforms.uMousePosition.value.copy(mouseUv);
  mesh.rotation.y = state.clock.elapsedTime * 0.35;
});

// 3. 绑到 material
<shaderMaterial
  vertexShader={vertexShaderCode}
  fragmentShader={fragmentShaderCode}
  uniforms={uniforms}
/>
React state (mouseUv) ↓ props useFrame 每帧 ↓ mat.uniforms.uTime.value = … Three.js 上传 GPU ↓ Fragment Shader 读 uniform uTime

关键：GLSL 里的 uniform float uTime 名字必须和 JS 里 uniforms.uTime 一致，否则传不进去。

9. 鼠标怎么进 Shader

文件：src/App.tsx

const x = (e.clientX - rect.left) / rect.width;   // 0~1 左→右
const y = 1 - (e.clientY - rect.top) / rect.height; // 0~1 下→上（与 UV 一致）
setMouseUv(new THREE.Vector2(x, y));

浏览器鼠标 Y 轴向下，而 UV 通常 Y 向上，所以要 1 - …。这样 Fragment 里的 vUv 和 uMousePosition 在同一坐标系，热点才对得上。

10. 动手实验（按顺序做）

实验 A · UV 可视化（30 秒）

在 fragmentShader.ts 的 main() 末尾改成：

gl_FragColor = vec4(vUv, 0.0, 1.0);

左下黑、右上红绿渐变 — 确认 Varying 插值正常。

实验 B · 只留时间动画

gl_FragColor = vec4(vec3(sin(uTime) * 0.5 + 0.5), 1.0);

整球同步明暗呼吸 — 确认 Uniform uTime 在动。

实验 C · 加大鼠标热点

把 dist * 6.0 改成 dist * 2.0，移动鼠标看光斑变大。

实验 D · 在 Vertex 里改形状

在 vertex shader 的 main() 里，gl_Position 之前加：

vec3 pos = position * (1.0 + 0.08 * sin(uTime * 3.0 + uv.y * 10.0));

并把 vec4(position, 1.0) 改成 vec4(pos, 1.0)（需在 vertex 也声明 uniform float uTime;）。

实验 E · 新增 Uniform

  1. JS：uIntensity: { value: 1.5 } 加进 uniforms
  2. GLSL：uniform float uIntensity;，乘到 finalColor 上
  3. 用 useFrame 或 React slider 改 value — 走通完整 CPU→GPU 链路
